# PRD · 尤克里里 AI 互动学习 App

> **文档版本**：v1.0
> **创建日期**：2026-06-26
> **对标产品**：AI 音乐学园（iMMmusician，北京音悦荚科技，2017 年上线，累计 3000 万+ 学员）
> **项目定位**：以尤克里里为核心切入乐器，自研一款"AI 互动 + 游戏化练习 + 海量曲谱"于一体的音乐学习 App，长期对标并逐步超越"AI 音乐学园"全部功能模块。

## 0. 文档说明

### 0.1 文档目的
本 PRD（产品需求文档）作为本项目开发的**总纲与方向锚点**，用于：
1. 明确产品要做什么、不做什么；
2. 划定功能边界与优先级（MVP → V1 → V2），指导分阶段开发；
3. 统一技术、设计、内容各方的认知基线，后续所有设计稿、技术方案、迭代计划均以本文档为准并持续回溯。

### 0.2 对标分析结论（"AI 音乐学园"成功要素）

| 维度 | AI 音乐学园做法 | 本项目借鉴/差异化策略 |
|------|----------------|----------------------|
| **核心技术** | 乐音识别（pitch detection）+ AI 互动视频教学 | 自研/集成开源音高识别（如 aubio / CREPE / Pitchy），先做"听得准"，再做强反馈 |
| **教学体验** | 还原线下 1 对 1，实时纠正音准节奏 | 复刻"实时听音→即时评分→可视化反馈"闭环 |
| **课程体系** | 必修 / 选修 / Pro 课，分乐器品类 | 先聚焦尤克里里单一品类做深做透，再横向扩展 |
| **学习辅助工具** | 调音器、节拍器、和弦库、节奏练习、五线谱认音 | 全部纳入，工具模块为高频刚需、留存抓手 |
| **练琴形式** | 游戏化练琴、自定义学习目标、打怪升级 | 引入"每日任务/成就/排行榜/连续打卡"游戏化机制 |
| **内容资产** | 海量吉他/钢琴/尤克里里谱 | 自建曲谱库 + 支持 MusicXML/ChordPro 导入，UGC 优先 |
| **商业模式** | 按品类永久会员 ¥199–359 / 年费订阅 / 全品类 VIP | 初期免费/低价切入，会员体系后置（见 §8） |
| **多平台** | iOS / Android / Windows | 本项目优先 **移动端（Flutter 跨平台 iOS+Android）**，后期扩展桌面/Web |

### 0.3 术语表

| 术语 | 含义 |
|------|------|
| **乐音识别（Pitch Detection）** | 通过麦克风实时采集音频，识别用户弹奏的音高、节奏 |
| **弹唱（Strum & Sing）** | 边弹和弦边演唱的演奏形式，尤克里里最主流玩法 |
| **指弹（Fingerstyle）** | 纯器乐独奏，旋律+伴奏在同一把乐器上完成 |
| **和弦图（Chord Diagram）** | 标注左手按弦位置、右手节奏型的指法图示 |
| **扫弦（Strumming）** | 右手节奏型弹奏方式 |
| **BPM** | Beats Per Minute，每分钟节拍数 |
| **MVP** | 最小可行产品，第一版可发布上线的最小功能集 |

## 1. 产品概述

### 1.1 产品愿景
> **让每一个想学尤克里里的人，都能拥有一位"听得懂你弹琴"的 AI 私教。**

### 1.2 产品定位
一款**面向尤克里里学习者（重点零基础~进阶用户）**的 AI 互动音乐教学 App，通过：
- 🎧 **实时乐音识别** 听用户弹奏并即时评分纠错；
- 🎬 **互动视频教学** 还原线下 1 对 1 私教体验；
- 🎮 **游戏化练琴** 让坚持学习不再枯燥；
- 📚 **海量曲谱 + 工具集** 满足日常练习与弹唱需求。

### 1.3 目标用户

| 用户画像 | 占比预估 | 核心诉求 | 痛点 |
|---------|---------|---------|------|
| **零基础小白**（学生/上班族，18–35 岁） | 50% | 低门槛入门、快速弹唱一首歌 | 看不懂谱、不知道弹得对不对、难坚持 |
| **自学进阶者**（已会基础和弦） | 30% | 系统提升技巧、突破瓶颈 | 缺乏反馈、节奏不稳、不知道下一步学什么 |
| **亲子/儿童启蒙** | 10% | 趣味启蒙、家长可监督 | 孩子注意力短、缺乏激励 |
| **尤克里里爱好者/演奏向** | 10% | 指弹、风格曲目、曲谱资源 | 高阶内容稀缺 |

### 1.4 核心价值主张
1. **听得准**：AI 实时识别弹奏，精准反馈音准与节奏；
2. **学得快**：体系化课程 + 互动视频，7 天弹唱第一首歌；
3. **练得爽**：游戏化机制 + 海量曲谱，让练琴上瘾；
4. **玩得起**：免费核心功能 + 低价会员，降低学习门槛。

## 3. 详细功能需求

> 说明：每个功能标注 **【优先级】** — P0（MVP 必备）/ P1（V1 重要）/ P2（V2 增强）/ P3（远期）。

> 💰 **权益标注**（商用预留）：为支撑后期会员体系，功能另按权益分层，详见 `docs/MONETIZATION.md` §2 权益矩阵。图标含义：
> - 🟢 `免费` = 注册即可用（学琴刚需工具，提升留存）
> - 🟡 `试用` = 试用期内全开，免费层限额（如每日 N 次）

> **MVP 阶段全部免费可用**（架构已预留 `FeatureGate` 权限层，见 `docs/TDD.md` §6.5），Phase 2 启用试用期与免费限额，Phase 3 上线会员售卖。

### 3.2 模块二：学习（互动课程）

#### 3.2.1 课程体系
对标 AI 音乐学园的**必修 / 选修 / Pro 课**三级结构：

| 课程类型 | 定位 | 内容方向（尤克里里） | 优先级 |
|---------|------|---------------------|--------|
| **必修课** | 体系化基础，从 0 到 1 | 持琴姿势 → 调音 → 基础和弦（C/Am/F/G）→ 扫弦节奏 → 第一首弹唱 → 指弹入门 | P0 |
| **选修课** | 风格/技巧专项 | 夏威夷风、流行弹唱、指弹独奏、技巧专项（打板/泛音/切音） | P2 |
| **Pro 课** | 进阶/专业 | 高阶指弹曲目精讲、乐理系统、改编编曲 | P3 |

#### 3.2.2 互动视频教学（核心）

**功能流程**：
```
进入课程 → 播放教学视频 → 老师示范 → "该你了"提示
   → 麦克风收音 → AI 实时识别 → 实时评分（音准/节奏）
   → 屏幕反馈（绿色✓正确/红色✗错误/进度条）
   → 不达标自动重练 → 通过进入下一段 → 课程完成发成就
```

**关键能力**：
| 能力 | 说明 | 优先级 |
|------|------|--------|
| 教学视频分段播放 | 按乐句/小节切分，示范与练习交替 | P0 |
| **AI 实时音准识别** | 麦克风采集，识别弹奏音高，对比标准音 | P0（技术核心） |
| 实时节奏评测 | 判断节拍是否对齐（基于 onset detection） | P1 |
| 可视化反馈 | 滚动曲谱 + 当前音符高亮 + 评分仪表盘 | P0 |
| 跟弹模式（Call & Response） | 老师弹一句→用户跟弹一句→评分 | P0 |
| 分段练习/AB 循环 | 难点乐句单独循环练习 | P1 |
| 速度自适应 | 支持降速到 50%–100% 练习 | P1 |
| 练习报告 | 课程结束后生成评分报告（音准/节奏/完成度） | P1 |

#### 3.2.3 学习路径与进度
- **技能树可视化**：展示已掌握/待学习技能节点（如和弦、节奏型）；
- **课程进度持久化**：记录每节课完成度、最高分、练习次数；
- **P1**：智能推荐下一节课。

### 3.3 模块三：练琴（工具集 + 游戏化）

#### 3.3.1 智能调音器 【P0 · MVP 核心】
| 子功能 | 说明 |
|--------|------|
| 麦克风实时收音 | 申请录音权限 |
| 音高检测 | 显示当前音高（如 G4）与偏差（cents） |
| 标准调音预设 | 尤克里里标准 G-C-E-A（含 High-G/Low-G 切换） |
| 视觉指针 | 指针表盘：偏左低/偏右高/居中绿色✓ |
| 自动弦序引导 | 依次引导 4 根弦调准 |
| 调音历史 | 记录最近调音（可选） |

#### 3.3.2 节拍器 【P0】
- BPM 调节（40–240）；
- 拍号选择（2/4、3/4、4/4、6/8）；
- 重音设置、音色选择、视觉闪烁；
- 节拍细分（八分/三连音/十六分）；
- **P2**：节拍器+弹奏同步录音回放。

#### 3.3.3 和弦库 【P0】
- 常用和弦指法图（C/Dm/Em/F/G/Am/Bm 等及变位）；
- 点击听和弦标准音色；
- 按和弦名搜索、按难度分类；
- 和弦变体（maj7、7、m、dim 等）；
- **P1**：和弦按Difficulty/调式筛选。

#### 3.3.4 和弦转换练习 【P1】
- 设定和弦进行（如 C-G-Am-F），练习转换速度；
- 计时/计次，记录平均转换时长；
- 难度递增（2 和弦→4 和弦）。

#### 3.3.5 节奏/扫弦练习 【P1】
- 内置常用扫弦节奏型（民谣下下上上下上等）；
- 跟随节拍视觉提示扫弦；
- 麦克风识别扫弦节奏是否吻合；
- 支持自定义节奏型。

#### 3.3.6 音准练习（跟弹） 【P1】
- 给定一段旋律/音阶，用户跟弹评分；
- 类似"打地鼠"游戏：音符落到判定线弹对得分。

#### 3.3.6a 整曲弹唱（节奏模式） 【P1 · 核心】
> 对标 AI 音乐学园核心体验。从"弹对才走"升级为"按节奏流水滚动"。

**两种练习模式：**

| 模式 | 面向 | UI 范式 | 判定方式 |
|------|------|---------|---------|
| **和弦伴奏版** | 弹唱扫弦 | 和弦名+指法图+歌词三行垂直对齐，从右向左滚动，判定线高亮 | Chroma 和弦识别 |
| **单音旋律版** | 指弹旋律 | 四线谱 TAB（4 弦各一行），数字标注品数，从右向左滚动 | NCCF 单音识别 |

**功能要点：**
- **配乐驱动**：按 BPM 自动播放配乐，曲谱按节奏流水滚动（不等用户）
- **判定线**：固定在屏幕中央，和弦/音符滚到判定线时为弹奏时刻
- **弹奏提示**：配乐同步 + 视觉高亮 + 音符逼近（双重提示，新手友好）
- **实时反馈**：弹对变绿，弹错标红，没弹到标灰，传送带不停
- **横屏**：整曲模式强制横屏，充分利用屏幕宽度
- **BPM 可调**：慢速 50 → 原速 80 → 快速 120，适配不同水平
- **完成报告**：准确率 + 弹对/总数 + EXP 奖励
- **指法图/品数**：和弦版显示指法图，单音版显示品数（新手不记得按法）

#### 3.3.7 五线谱认音练习 【P2】
- 识谱小游戏，训练读谱能力。

#### 3.3.8 游戏化闯关练习 【P2】
- 关卡制（每关练习一个技巧点）；
- 评分星级、过关解锁；
- 限时挑战、连击得分。

### 3.4 模块四：曲谱

#### 3.4.1 曲谱库 【P0】
| 子功能 | 说明 |
|--------|------|
| 曲目列表 | 按热度/难度/风格/标签筛选排序 |
| 难度分级 | 入门 ⭐ / 进阶 ⭐⭐⭐ / 指弹 ⭐⭐⭐⭐⭐ |
| 搜索 | 按歌名/歌手/和弦标签搜索 |
| 详情页 | 尤克里里谱：和弦图 + 歌词 + 节奏型 + 调号 |

#### 3.4.2 曲谱播放器 【P0 · 核心】
- **滚动自动播放**：曲谱随播放进度滚动，当前小节高亮；
- **节拍器同步**：播放时叠加节拍器声；
- **变速/变调**：降速练习、变调适配嗓音；
- **AB 循环**：选段循环练习；
- **跟弹模式**：开启麦克风，AI 实时评分（与互动课程同源能力）。

#### 3.4.3 曲谱管理
- 收藏、最近浏览、我的谱（P1）；
- 离线下载（P1）；
- **P3**：用户上传曲谱（UGC）、曲谱导入（支持 MusicXML / ChordPro）。

## 4. 关键技术需求（PRD 层面约束）

### 4.1 核心技术难点：实时乐音识别
这是整个产品的**技术心脏**，直接决定体验上限。

| 指标 | 目标 |
|------|------|
| 音高识别延迟 | **< 100ms**（用户弹奏到看到反馈） |
| 音高准确度 | 误差 **< ±5 cents**（半音的 5%） |
| 支持识别范围 | 尤克里里音域 C3–C6（含 High-G/Low-G） |
| 环境抗噪 | 普通室内环境、手机麦克风可用 |
| 节奏识别 | onset 检测，节拍对齐误差 **< 50ms** |

**候选技术方案**（技术调研阶段确定）：
- 移动端实时：`Pitchy`（JS）/ `TarsosDSP`（Java）/ Flutter 的 `pitch_detector` 插件；
- 高精度离线：`CREPE`（TensorFlow）、`aubio`（C，性能优）；
- 架构：本地实时识别优先（低延迟），复杂评测可上云。

### 4.2 平台与技术栈（建议）

| 层 | 选型建议 | 理由 |
|----|---------|------|
| **移动端框架** | **Flutter**（Dart） | 一套代码 iOS + Android，AI 音乐学园多端覆盖，跨平台性价比高 |
| **音频处理** | Flutter + 原生通道（iOS AVAudioEngine / Android AudioRecord） | 实时音频采集需高性能原生实现 |
| **音高识别** | 本地 `pitch_detector_plus` 或自研原生插件 | 低延迟 |
| **后端** | Node.js / Python（FastAPI） | 课程/曲谱/用户数据服务 |
| **数据库** | MySQL（业务）+ Redis（缓存/排行榜） | 常规组合 |
| **存储** | 对象存储（视频/音频/曲谱文件） | 阿里云 OSS / 七牛 |
| **AI 模型**（可选） | 高阶评分模型后期上云 | 初期规则+本地识别足够 |

> ⚠️ 技术栈为 PRD 阶段建议，最终以技术方案评审为准。

### 4.3 内容生产需求
- **课程内容**：需录制教学视频 + 编写互动脚本（分段、评分点）；
- **曲谱内容**：需制作尤克里里谱数据（和弦、歌词、节奏型），建议采用结构化格式（如自定 JSON Schema 或 ChordPro）；
- **冷启动内容量**（MVP 目标）：≥ 30 节必修课 + ≥ 100 首热门弹唱曲目。

## 8. 商业模式设计（参考）

借鉴 AI 音乐学园"分品类永久会员 + 订阅 + 全品类 VIP"模式，本项目初期以**免费+低价**切入：

## 9. 风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| **乐音识别精度不达标** | 核心体验受损 | MVP 阶段集中攻关，多方案对比，预留云端高精度识别兜底 |
| **音频延迟过高** | 交互卡顿、评分不准 | 本地原生采集+识别，优化音频缓冲 |
| **课程/曲谱内容版权** | 合规风险、下架 | 优先公有领域曲目；UGC 版权审核；与合作方授权 |
| **内容生产成本高** | 上线节奏慢 | 结构化曲谱工具提效；初期聚焦精选曲目；预留 UGC 补充 |
| **用户留存难** | 流失快 | 游戏化机制、打卡、社交激励前置到 V1 |
| **对标产品先发优势** | 获客难 | 差异化（低价/更好的反馈/垂直尤克里里深度）+ 社区运营 |

### 10.2 尤克里里基础知识备忘
- **标准调音**：G4–C4–E4–A4（从 4 弦到 1 弦）；High-G（最常见）与 Low-G 两种配置；
- **常用入门和弦**：C、Am、F、G、Em、Dm（前 4 个即可弹唱大量歌曲）；
- **常见调号**：C 调、G 调为主流弹唱调；
- **尺寸**：21 寸（Soprano）/23 寸（Concert）/26 寸（Tenor），影响音域与音色。
